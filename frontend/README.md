# Frontend — Dispatch (Next.js 14 + TypeScript)

Newspaper-style news site built with Next.js App Router. Deployed to **Vercel**.

## Stack

- Next.js 14 (App Router, React Server Components, ISR)
- TypeScript (strict)
- Tailwind CSS (custom `ink` / `brand` palette)
- TipTap rich-text editor
- `isomorphic-dompurify` to sanitize rendered HTML
- `next/font` (Source Serif 4 + Inter)

## Local setup

```bash
cd frontend
cp .env.local.example .env.local
# set NEXT_PUBLIC_API_URL + NEXT_PUBLIC_SITE_URL
npm install
npm run dev
```

Open http://localhost:3000.

## Environment variables

All variables are prefixed `NEXT_PUBLIC_` because they are read in the browser.

| Variable                 | Required | Description                                                  |
| ------------------------ | -------- | ------------------------------------------------------------ |
| `NEXT_PUBLIC_API_URL`    | **yes**  | Backend origin, e.g. `https://api.yourdomain.com`            |
| `NEXT_PUBLIC_SITE_URL`   | **yes**  | Public URL of this frontend (used in sitemap.xml + SEO)      |
| `NEXT_PUBLIC_SITE_NAME`  | no       | Masthead name. Default `Dispatch`.                           |

## Routes

| Route                | Access    | Purpose                                                        |
| -------------------- | --------- | -------------------------------------------------------------- |
| `/`                  | public    | Homepage — hero, top stories, trending strip, category strips  |
| `/category/:slug`    | public    | Category landing (World, Business, Markets, …)                 |
| `/blog/:slug`        | public    | Full article with sanitized HTML + JSON-LD `NewsArticle`       |
| `/search`            | public    | Fuzzy search (typo-tolerant, powered by Fuse.js on the server) |
| `/login`             | public    | Sign-in form                                                   |
| `/register`          | public    | Registration form                                              |
| `/write`             | author    | Rich-text editor                                               |
| `/edit/:id`          | owner/admin | Edit an existing story                                       |
| `/dashboard`         | author    | Published + drafts with inline edit / delete                   |
| `/admin`             | admin     | Moderation console (stats, posts, users)                       |
| `/sitemap.xml`       | public    | Auto-generated, revalidated hourly                             |
| `/robots.txt`        | public    | Disallows `/dashboard`, `/write`, `/edit`, `/admin`, `/search` |

## Auth

- **Access token** (short-lived) kept in module memory — not `localStorage`, not `document.cookie`. Not exposed to XSS-reachable storage.
- **Refresh token** (long-lived) managed entirely by the browser as an httpOnly cookie set by the backend. JS cannot read it.
- On any 401 from a bearer-authenticated call, `lib/api.ts` transparently POSTs `/api/auth/refresh` (the cookie is auto-attached) and retries once.
- On app boot, `AuthContext` tries `/refresh` → `/me` to restore a session.

This means `credentials: 'include'` is set on every fetch, and the backend must list the frontend's exact origin in `CORS_ORIGIN` (cookies and `*` cannot coexist).

## Deploying to Vercel

1. Push the repo to GitHub.
2. In Vercel: **New Project**, import the repo.
3. **Root Directory**: `frontend`
4. **Framework preset**: Next.js (auto-detected).
5. Add env vars:
   - `NEXT_PUBLIC_API_URL=https://<your-backend>.onrender.com`
   - `NEXT_PUBLIC_SITE_URL=https://<your-project>.vercel.app`
6. Deploy.

After first deploy, take the Vercel URL and set it as `CORS_ORIGIN` and `PUBLIC_SITE_URL` on the backend.

## Performance & SEO

- **ISR** on all public read pages (`revalidate: 15–60s`) — Vercel caches the HTML and revalidates in the background.
- **`next/image`** with AVIF/WebP, blur-less bg placeholder, automatic `srcset`.
- **Sitemap + robots** generated by `app/sitemap.ts` and `app/robots.ts` (revalidated hourly).
- **SEO tags per article**: canonical URL, OG image, Twitter card, JSON-LD `NewsArticle`.
- **Client sanitization**: TipTap schema only allows safe tags; on rendering, `DOMPurify` strips anything suspect a second time before `dangerouslySetInnerHTML`.
